# 事件系统

事件系统位于 `core/event.h`，当前是轻量级 pointer 事件和交互状态封装。

## 1. PointerEvent

```cpp
struct PointerEvent {
    double x;
    double y;
    double deltaX;
    double deltaY;
    bool down;
};
```

含义：

- x / y：当前鼠标坐标
- deltaX / deltaY：和上次读取的差值
- down：鼠标左键是否按下

读取方式：

```cpp
core::PointerEvent event = core::readPointerEvent(window, pointerScale);
```

`pointerScale` 用来把 GLFW window 坐标转换成 framebuffer 坐标。

## 2. 为什么需要 pointerScale

当前渲染坐标使用 framebuffer 像素。  
GLFW cursor position 通常是 window 坐标。

所以 hit-test 前需要转换：

```cpp
x *= pointerScale;
y *= pointerScale;
```

注意：`pointerScale` 和 `dpiScale` 不一定相同。

- `dpiScale`：系统内容缩放，用于 UI 尺寸和字号
- `pointerScale`：鼠标坐标到 framebuffer 坐标的比例

## 3. InteractionState

```cpp
struct InteractionState {
    bool hover;
    bool pressed;
    bool clicked;
    bool drag;
    bool changed;
    double dragStartX;
    double dragStartY;
    double dragDeltaX;
    double dragDeltaY;

    void update(const Rect& bounds, const PointerEvent& event);
};
```

状态含义：

- hover：指针是否在 bounds 内
- pressed：鼠标是否在当前组件上按下
- clicked：本帧是否完成点击
- drag：是否正在拖动
- changed：交互状态是否变化

## 4. 点击逻辑

当前点击判定：

```cpp
clicked = hover && pressed && !event.down;
```

也就是：

1. 鼠标在组件上
2. 前一状态是 pressed
3. 当前鼠标松开

这表示在组件上完成一次 click。

## 5. Hover 与 Pressed

```cpp
hover = bounds.contains(event.x, event.y);
pressed = hover && event.down;
```

当前 pressed 只有在 hover 内按下才为 true。  
如果按下后拖出组件，pressed 会变 false。

这对 Button 合适，但对 Slider / Drag Handle 可能不够。后续需要 pointer capture。

## 6. Drag

当前 drag 判定：

```cpp
drag = pressed && (abs(dragDeltaX) > 2.0 || abs(dragDeltaY) > 2.0);
```

拖动阈值是 2 像素，避免轻微抖动被判为 drag。

## 7. Button 如何使用事件系统

`components::Button` 内部持有：

```cpp
core::InteractionState interaction_;
```

更新时：

```cpp
interaction_.update(bounds_, core::readPointerEvent(window, pointerScale));
```

然后用：

- `interaction_.hover`
- `interaction_.pressed`
- `interaction_.clicked`
- `interaction_.drag`

驱动按钮动画和状态。

## 8. 当前限制

- 只有鼠标左键
- 没有键盘事件
- 没有 focus
- 没有 pointer capture
- 没有事件冒泡 / 捕获
- 没有组件树命中测试
- 旋转/缩放后的组件仍按原始 bounds hit-test

后续 GUI 框架化时，需要增加：

- Widget Tree
- HitTestResult
- Event Dispatch
- Capture / Bubble
- Focus Manager
- Pointer Capture
