# 渲染流程

本文档说明当前程序的通用入口、渲染循环、DPI 缩放、事件等待和 app/demo 的职责边界。

## 1. 文件职责

当前渲染入口在 `main.cpp`。

`main.cpp` 只负责框架级工作：

- 初始化 GLFW / GLAD
- 创建 OpenGL 3.3 Core 窗口
- 开启 VSync：`glfwSwapInterval(1)`
- 监听 framebuffer resize / refresh / content scale
- 计算 DPI scale 和 pointer scale
- 调用 app 生命周期
- 控制按需渲染和动画期间的事件等待

具体画什么不在 `main.cpp` 里写，而是在 `app/*.cpp` 中实现。

`app/app.h` 定义 app 需要实现的接口：

```cpp
namespace app {

const char* windowTitle();
bool initialize(GLFWwindow* window);
bool update(GLFWwindow* window,
            float deltaSeconds,
            int windowWidth,
            int windowHeight,
            float dpiScale,
            float pointerScale);
bool isAnimating();
void render(int windowWidth, int windowHeight, float dpiScale);
void shutdown();

}
```

## 2. 主循环

主循环核心逻辑：

```text
计算 deltaSeconds
处理 ESC 退出
读取 framebuffer 尺寸
读取 dpiScale / pointerScale
调用 app::update(...)
如果需要重绘:
    app::render(...)
    glfwSwapBuffers(...)
如果 app 正在动画:
    按显示器刷新周期 sleep
    glfwPollEvents()
否则:
    glfwWaitEvents()
```

这样做的目标是：

- 空闲时不持续占 CPU
- UI 状态变化时才重绘
- 动画中才短时间连续刷新
- 动画刷新节奏跟显示器刷新率接近

## 3. 按需渲染

`WindowState` 里只有一个关键标记：

```cpp
struct WindowState {
    bool needsRender = true;
};
```

这些情况会让 `needsRender = true`：

- 首帧
- framebuffer 尺寸变化
- 窗口 refresh
- 系统 content scale 变化
- `app::update(...)` 返回 true

组件侧例如 Button 会在 hover / pressed / click / 动画值变化时返回需要渲染。

## 4. DPI Scale 和 Pointer Scale

当前有两个 scale，不要混用。

`dpiScale` 来自：

```cpp
glfwGetWindowContentScale(window, &scaleX, &scaleY);
```

用途：

- UI 逻辑尺寸转 framebuffer 像素
- 字号缩放
- 圆角、阴影、边框等视觉尺寸缩放

`pointerScale` 来自：

```cpp
framebufferSize / windowSize
```

用途：

- 把 GLFW cursor position 转成 framebuffer 坐标
- 给 hit-test 使用

原因是系统内容缩放和鼠标坐标到 framebuffer 的实际比例不一定完全相同。

## 5. VSync 与动画节奏

程序开启了：

```cpp
glfwSwapInterval(1);
```

动画期间还会读取显示器刷新率：

```cpp
glfwGetVideoMode(monitor)->refreshRate
```

然后用对应 frame interval 做 pacing。  
这不是把动画写死为 60 FPS，而是跟当前显示器刷新率走。

## 6. app/demo 当前结构

`app/demo.cpp` 当前分为三块：

- `DemoMetrics`：逻辑尺寸、字号、圆角等设计值
- `configureStyles()`：颜色、渐变、阴影、边框等静态样式
- `syncLayout()`：根据 layout 结果和 DPI 设置组件 frame/fontSize

这让 demo 不再散落写固定像素位置，也避免 initialize 里设置字号、layout 里又覆盖字号的混乱。
